---

# Project Overview

E-commerce platforms generate huge volumes of transaction data daily. Predicting customer demand helps businesses:

• Reduce stock shortages
• Optimize inventory management
• Improve supply chain efficiency
• Increase operational profitability

This project builds an **AI-driven forecasting system** that predicts customer order quantities using the **Brazilian Olist E-Commerce dataset**.

Main components:

• Azure Data Factory (Data Pipelines)
• Azure Blob Storage (Data Storage)
• Machine Learning (XGBoost Regression)
• Power BI (Dashboard Visualization)

---

# System Architecture

The project follows a **modern cloud data pipeline architecture**.

Raw Data (CSV Files)
↓
Azure Blob Storage
↓
Azure Data Factory (Copy Activity)
↓
ADF Mapping Data Flow (Transformation)
↓
Curated Dataset
↓
Machine Learning Model (XGBoost)
↓
Predictions Generated
↓
ADF Pipeline Automation
↓
Power BI Dashboard

This architecture integrates **data engineering, machine learning, and analytics** in one automated pipeline.

# Data Engineering Pipeline

## Data Ingestion

Raw CSV datasets are uploaded into **Azure Blob Storage**.

Azure Data Factory **Copy Activity** performs ingestion.

Key features:

• Serverless data ingestion
• Scalable pipeline design
• Raw data preserved in staging layer

---

## Data Transformation

Azure Data Factory **Mapping Data Flow** performs transformations.

Operations performed:

• Dataset joins
• Column selection
• Data cleaning
• Schema standardization
• Feature selection

The result is a **single curated dataset for ML training**.

# Machine Learning Model

Model used: **XGBoost Regressor**

Training environment: **Google Colab**

Steps performed:

1 Load curated dataset
2 Perform feature selection
3 Encode categorical variables
4 Train regression model
5 Evaluate model performance

Evaluation metrics:

MAE (Mean Absolute Error)
RMSE (Root Mean Square Error)

Example output:

R² Score ≈ 0.73
RMSE ≈ 6.12

---

# Prediction Pipeline Automation

Azure Data Factory pipelines automate prediction delivery.

Pipeline workflow:

1 Predictions generated by ML model
2 Stored in Azure Blob Storage staging folder
3 Copy activity moves predictions to latest folder
4 Historical predictions archived automatically
5 Staging files deleted after execution

Benefits:

• Fully automated system
• Reliable data pipeline
• Versioned prediction history
